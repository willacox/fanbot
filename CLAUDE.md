# CLAUDE Project Rules

You MUST follow these rules at all times for this project.

## Core Principles
- Prioritize clarity, maintainability, and correctness.
- Never assume undocumented behavior.
- Re-read relevant files before making changes.

## Architecture
- Follow established architectural patterns.
- Keep domain logic isolated from infrastructure.
- Avoid tight coupling between modules.

## Coding Standards
- Use explicit types.
- Write small, composable functions.
- Avoid hidden side effects.

## Working Protocol
- Always read /memory-bank files before starting work.
- If context is unclear, stop and ask.
- Update memory-bank files at the end of each session.
